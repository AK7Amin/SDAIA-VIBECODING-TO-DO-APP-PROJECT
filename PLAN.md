## المرحلة ٢ — ترقية قاعدة البيانات والـ API للميزات الجديدة (TDD)

- **مخطط جديد لجدول المهام:** إضافة أعمدة
  `due_date TEXT` (تاريخ استحقاق ISO: yyyy-mm-dd)،
  `priority INTEGER` (1=منخفضة، 2=عادية، 3=عالية)،
  `created_at TEXT`.
- **ترحيل (migration) آمن:** فحص `PRAGMA table_info` وإضافة الأعمدة الناقصة
  بـ `ALTER TABLE ADD COLUMN` — بدون لمس البيانات الموجودة.
- **الـ API:** إنشاء/تعديل المهمة يقبل `due_date` و`priority` مع تحقق صارم
  (تاريخ غير صالح → 400، أولوية خارج 1-3 → 400)؛ الإرجاع مرتب:
  غير المنجزة أولاً ← الأولوية الأعلى ← الاستحقاق الأقرب.
- **اختبارات:** التحقق، الترتيب، وأن حماية IDOR ما زالت تشمل الحقول الجديدة.

**⚠️ بوابة مراجعة بشرية:** ترحيل مخطط قاعدة بيانات (CLAUDE.md §6.2).

## المرحلة ٣ — تحسين الواجهة

- **إعادة هيكلة:** فصل CSS/JS إلى `static/` (يمهد لـ CSP صارم `default-src 'self'` — يُضاف هنا).
- **تصميم أجمل:** خط أنظف، ألوان متناسقة، وضع RTL/عربي اختياري، استجابة للجوال.
- **عناصر الميزات الجديدة:**
  - حقل تاريخ استحقاق + قائمة أولوية في نموذج الإضافة
  - شارة تاريخ على كل مهمة: أحمر متأخرة، برتقالي اليوم، رمادي قادمة
  - شريحة لون للأولوية (عالية/عادية/منخفضة)
  - فلاتر أعلى القائمة: الكل / نشطة / منجزة / متأخرة + عدّادات
  - تعديل بنافذة صغيرة بدل التعديل السطري (عنوان + تاريخ + أولوية معاً)
- **تحقق في المتصفح** بعد كل جزء، وتحديث صور README والـ GIF في النهاية.
